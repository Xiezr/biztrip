# Biztrip 差旅日历 — 产品需求文档 (PRD)

> 版本：v1.0 | 生成日期：2026-05-24 | 基于代码反向梳理

---

## 一、产品定位

Biztrip 是一款面向差旅频繁用户的 **Android 差旅日历 APP**，核心价值为：
- 以**日历涂抹**的直观方式记录差旅
- 覆盖差旅**全生命周期**的自动提醒（出发前准备 → 结束前跟进 → 结束後报销/报告）
- 目的地**可复用、可归档**，历史数据永久保留

---

## 二、 personas / 目标用户

| 角色 | 痛点 | 本产品解法 |
|---|---|---|
| 商务差旅人士 | 差旅频繁，容易忘记报销/报告截止日期 | 自动跟进提醒 |
| 项目人员 | 多目的地、多时间段难以管理 | 日历涂抹 + 目的地复用 |
| 财务合规用户 | 发票/单据容易丢失 | 拍照存档 + 本地持久化 |

## 五、技术架构摘要

```
数据层：  TravelLocation (模型) + TravelMark (标记) + StorageService (持久化)
状态层：  CalendarProvider (视图状态) + LocationProvider (目的地 CRUD) + MarkProvider (标记管理) + NotificationService (通知计算)
UI 层：   YearView / MonthView / TwoMonthView / LocationEditPage / NotificationListPage
原生层：  MainActivity.kt (MethodChannel: 相机 + 位置权限)
```

## 六、结论

**已实现功能覆盖率：约 95%（核心功能）**

所有代码中存在的功能点均已确认实现。唯一的核心缺口是 **?1 通知实际推送**——当前通知系统只计算通知内容并展示在应用内列表，未调用 Android `NotificationManager` 向系统通知栏推送。

建议下一步优先实现 ?1，使通知真正触达用户。
